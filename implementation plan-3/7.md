The month field in rent_payments currently stores a free-text string (e.g. "July 2026" based on the placeholder). For the Task 4 YYYY-MM comparison to work reliably, the quick-action button will prefill month as "2026-07" (ISO format). Existing manually entered rows with "July 2026" text won't match. I'll add a note in the UI near the month field suggesting ISO format, but won't break existing manual entries. Is this acceptable, or do you want to normalize existing data?
